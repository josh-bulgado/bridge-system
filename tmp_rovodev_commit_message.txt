- Store files in 1MB chunks with metadata (contentType, originalFileName)

Frontend:

- Integrate file upload with VerificationPage form

Features:
- No external storage needed (S3, Azure Blob)
- Files stored directly in MongoDB using GridFS
- Automatic file replication with MongoDB
- Visual feedback (idle, dragging, uploading, uploaded states)
- One-click file removal
- Full TypeScript type safety
- RESTful API endpoints (/api/FileUpload)

Storage:
- Files chunked into 1MB pieces for efficient streaming
- Metadata stored separately from file data
- URL format: /api/FileUpload/{objectId}
- Files can be downloaded via GET request
- Admin/staff can delete files via DELETE request
